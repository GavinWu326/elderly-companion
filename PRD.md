# 老人陪伴助手 PRD（产品需求文档）

## 文档信息
- **产品名称**: 老人陪伴助手
- **版本**: v1.0
- **文档日期**: 2026-03-10
- **文档类型**: 产品需求文档（PRD）

---

## 1. 产品概述

### 1.1 产品定位
一款以语音交互为核心的老人陪伴与生活助手应用，让老年人"开口就能用"，提供日常陪伴、生活提醒、应急引导等核心功能，同时支持家属追溯关键记录。

### 1.2 产品愿景
降低老年人独居时的孤独感与生活风险门槛，让语音成为最自然的交互方式，实现"像孙子孙女一样陪聊、关键时刻能救命"的体验。

### 1.3 核心价值
- **情感陪伴**: 日常聊天、倾听、情绪支持
- **生活助手**: 提醒、问答、信息查询
- **安全保障**: 应急引导、风险识别、求助路径
- **家庭连接**: 记录追溯、家属关怀

### 1.4 运行形式
- **平台**: 手机Web端（浏览器访问）
- **部署**: 本地部署
- **交互方式**: 语音为主，文字为辅

## 2. 目标用户分析

### 2.1 核心用户群体
| 用户类型 | 年龄范围 | 使用场景 | 核心需求 |
|---------|---------|---------|---------|
| 主要用户 | 60岁以上 | 独居/半独居日常 | 陪伴、提醒、应急求助 |
| 辅助用户 | 40-60岁 | 家属/子女 | 追溯记录、了解状态 |
| 管理用户 | - | 后台管理员 | 数据查看、异常监控 |

### 2.2 用户画像

#### 主要用户画像 - 张大爷
- **年龄**: 72岁
- **居住情况**: 独居
- **身体状况**: 高血压、需要定时服药
- **技术能力**: 会接打电话、会用微信语音，不会打字搜索
- **主要痛点**: 
  - 子女工作忙，日常交流少
  - 经常忘记吃药、复诊
  - 身体不舒服时不知道怎么办
  - 看不清小字，操作复杂

### 2.3 用户痛点分析

#### 2.3.1 陪伴不足
- **问题**: 缺少日常交流与情绪价值
- **影响**: 孤独感强，情绪低落
- **需求**: 能聊天、能倾听、有回应

#### 2.3.2 记忆/执行力下降
- **问题**: 吃药、复诊、喝水等容易遗忘
- **影响**: 健康风险增加
- **需求**: 主动提醒、语音播报

#### 2.3.3 信息获取门槛高
- **问题**: 不会搜索、看不清小字、流程复杂
- **影响**: 信息闭塞，生活不便
- **需求**: 语音问答、简单交互

#### 2.3.4 突发风险应对弱
- **问题**: 肚子痛/头痛/胸痛等场景需要明确指引
- **影响**: 错过最佳处理时机
- **需求**: 应急引导、求助路径、留痕

## 3. 产品目标

### 3.1 核心目标
1. **陪伴与问答**: 语音对话稳定可用、支持连续多轮交流
2. **应急引导**: 识别高风险表达并进入"紧急模式"，提供清晰求助路径
3. **基础画像**: 支持最小化、可控的用户偏好与紧急联系人信息
4. **可追溯**: 对话、提醒/应急事件形成记录，家属/后台可查询
5. **合规与安全**: 不做医疗诊断；对高风险内容优先求助流程

### 3.2 量化指标
- 语音识别准确率 ≥ 85%
- 对话响应时间 ≤ 3秒
- 应急模式响应时间 ≤ 1秒
- 提醒准时率 ≥ 95%

### 4.2 功能详细说明

#### 4.2.1 用户中心

**功能描述**: 用户注册、登录、基础信息管理

**主要功能**:
1. **用户注册/登录**
   - 支持手机号+验证码登录
   - 支持微信授权登录（可选）
   - 简化的注册流程（仅需手机号）

2. **基础信息管理**
   - 姓名（可选）
   - 年龄（必填，用于适老化调整）
   - 主要健康状况（高血压/糖尿病/心脏病等，必填）
   - 常用药物（可选）
   - 语音偏好（男声/女声，可选）

3. **紧急联系人配置**
   - 联系人姓名（必填）
   - 联系方式（电话，必填）
   - 与老人关系（必填）
   - 可配置多个紧急联系人

**界面要求**:
- 大字体（≥18px）
- 高对比度
- 简化操作流程
- 支持语音输入

**数据要求**:
- 用户信息加密存储
- 紧急联系人实时同步

---

#### 4.2.2 陪伴对话

**功能描述**: 核心功能，提供日常聊天、倾听、情绪支持

**主要功能**:
1. **语音识别**
   - 按住说话，松开发送
   - 自动识别语音转文字
   - 支持方言优化（优先普通话，后续扩展）
   - 识别结果展示（可校正）

2. **智能对话**
   - 日常聊天：问候、闲聊、倾听
   - 情绪识别：识别负面情绪并安抚
   - 多轮对话：支持上下文记忆
   - 话题引导：天气、新闻、往事回忆等

3. **语音合成**
   - 自然语音播报
   - 支持音量、语速调整
   - 男女声切换
   - 情感化语音（高兴、安慰、严肃等）

**对话场景示例**:
```
用户: "今天天气怎么样？"
助手: "今天北京晴天，温度18-25度，适合出去走走哦。"

用户: "我有点难过"
助手: "怎么了？您愿意跟我说说吗？我在听呢。"

用户: "提醒我8点吃药"
助手: "好的，我记住了，晚上8点会提醒您吃药。"
```

**界面要求**:
- 语音按钮醒目（居中、大尺寸）
- 对话气泡清晰
- 自动播放语音回复
- 支持文字模式（备选）

**技术要求**:
- 语音识别准确率 ≥ 85%
- 响应时间 ≤ 3秒
- 支持连续对话上下文
- 情绪识别准确率 ≥ 70%

---

#### 4.2.3 生活提醒

**功能描述**: 服药、复诊、喝水等生活提醒功能

**主要功能**:
1. **服药提醒**
   - 添加提醒：药物名称、时间、剂量、频率
   - 到点提醒：语音播报+弹窗
   - 确认服药：用户确认后标记完成
   - 逾期提醒：未确认后15分钟再次提醒

2. **复诊提醒**
   - 添加提醒：医院、时间、科室
   - 提前提醒：提前1天、当天提醒
   - 路线指引（可选）

3. **喝水提醒**
   - 默认设置：每2小时提醒一次
   - 自定义间隔
   - 累计统计

4. **自定义提醒**
   - 添加任意提醒事项
   - 设置时间和内容

**提醒交互流程**:
1. 到点提醒 → 语音播报："该吃药了，记得吃[药名]"
2. 弹窗显示提醒详情
3. 用户点击"已服用" → 标记完成
4. 逾期未处理 → 15分钟后再次提醒

**界面要求**:
- 提醒列表清晰展示
- 添加/编辑/删除提醒
- 提醒历史记录

---

#### 4.2.4 应急引导

**功能描述**: 识别高风险表达，进入紧急模式，提供求助路径

**主要功能**:
1. **风险识别**
   - 关键词识别：
     - 身体不适：痛、难受、不舒服、晕、心慌、胸闷、喘不过气等
     - 危险表述：救命、救我、摔倒、起不来等
   - 优先级判定：
     - 高风险（救命、摔倒）：立即进入紧急模式
     - 中风险（身体不适）：询问具体情况
     - 低风险（一般疼痛）：建议休息+观察

2. **紧急模式**
   - 界面切换：红色主题、大按钮
   - 语音播报："检测到您可能遇到紧急情况，正在提供帮助"
   - 一键呼叫：拨打紧急联系人电话（需用户授权）
   - 引导对话：
     ```
     "您现在哪里不舒服？"
     "您现在能站起来吗？"
     "您现在是一个人在家吗？"
     ```
   - 求助路径：
     - 优先：拨打120/紧急联系人
     - 备选：指导自救措施
     - 通知：记录应急事件、通知家属

3. **应急记录**
   - 自动记录：时间、情况描述、处理方式
   - 通知家属：短信/微信通知（需要配置）

**应急引导流程**:
```
用户说"我胸口难受"
↓
系统识别为中风险
↓
语音询问："您胸口哪里难受？是闷还是疼？"
↓
用户回答
↓
系统判断是否需要紧急处理
↓
提供对应建议或进入紧急模式
```

**界面要求**:
- 紧急模式：红色背景、超大按钮
- 一键呼叫按钮
- 清晰的文字指引
- 支持语音引导

**安全要求**:
- 不做医疗诊断
- 明确声明：建议就医
- 高风险优先求助流程

---

#### 4.2.5 信息查询

**功能描述**: 天气、新闻、常识等生活信息查询

**主要功能**:
1. **天气查询**
   - 今日天气：温度、天气状况
   - 明日预报：温度、降雨概率
   - 出行建议：适合外出吗？需要带伞吗？

2. **新闻简报**
   - 简要播报3-5条重要新闻
   - 关注老年人兴趣：健康、民生、养生

3. **常识问答**
   - 常见问题：时间、日期、节日
   - 生活常识：饮食建议、运动建议
   - 健康知识：基础科普（非医疗建议）

**交互示例**:
```
用户: "今天天气怎么样？"
助手: "今天北京晴天，温度18-25度，适合出去走走。"

用户: "有什么新闻？"
助手: "今天有几条重要新闻：第一条...第二条...第三条..."
```

**家属端功能**:

4. **即时反馈**: 每个操作都有明确反馈
5. **容错性强**: 支持撤销、重新输入

### 5.3 语音交互流程

#### 5.3.1 日常对话流程
```

→ 开始录音
→ 用户松开
→ 语音识别
→ 生成回复
→ 语音合成

```

#### 5.3.2 应急模式流程
```
用户说"我难受"

→ 语音询问具体情况
→ 根据回答判断是否需要紧急处理
→ 提供对应建议或进入紧急模式
```

- 最小尺寸：60px × 60px
- 间距：≥16px
- 圆角：8px

#### 5.4.4 操作简化
- 减少操作步骤（最多3步）
- 提供语音替代操作
- 清晰的操作提示

## 7. 技术架构

### 7.1 技术栈

#### 前端
- **框架**: Vue 3
- **UI组件库**: TDesign Vue（企业级设计系统）
- **语音识别**: Web Speech API
- **语音合成**: Web Speech API
- **HTTP客户端**: Axios
- **状态管理**: Pinia

#### 后端
- **框架**: Node.js + Express
- **数据库**: SQLite（本地部署）
- **对话引擎**: OpenAI API / 本地LLM
- **语音服务**: 可选用腾讯云语音服务

#### 部署
- **本地部署**: Docker Compose
- **数据库**: SQLite
- **反向代理**: Nginx（可选）

## 8. 用户流程

### 8.1 新用户首次使用流程

```
1. 打开浏览器访问应用
   ↓
2. 输入手机号 → 获取验证码 → 登录
   ↓
3. 填写基础信息（姓名、年龄、健康状况）
   ↓
4. 配置紧急联系人（至少1个）
   ↓
5. 选择语音偏好（男声/女声）
   ↓
6. 完成设置，进入首页
   ↓
7. 语音引导："您好！我是您的陪伴助手，您可以按住按钮跟我说话"
```

### 8.2 日常对话流程

```
1. 用户按住"按住说话"按钮
   ↓
2. 开始录音（松开即发送）
   ↓
3. 语音识别 → 转文字
   ↓
4. 发送到后端 → 生成回复
   ↓
5. 语音合成 → 自动播放
   ↓
6. 显示对话气泡
```

### 8.3 添加提醒流程

```
1. 进入"提醒"页面
   ↓
2. 点击"添加提醒"
   ↓
3. 选择提醒类型（服药/复诊/喝水/自定义）
   ↓
4. 填写提醒信息
   ↓
5. 设置提醒时间
   ↓
6. 保存提醒
```

### 8.4 应急处理流程

```
1. 用户说"我难受/救命/摔倒"
   ↓
2. 系统识别关键词
   ↓
3. 判断风险等级
   ↓
4a. 高风险（救命/摔倒）
   → 立即进入紧急模式
   → 界面变红
   → 语音播报："检测到紧急情况，正在提供帮助"
   → 显示一键呼叫按钮

4b. 中风险（身体不适）
   → 语音询问具体情况
   → 根据回答判断
   → 提供建议或进入紧急模式

5. 自动记录应急事件
   ↓
6. 通知家属（如已配置）
```

### 8.5 家属查看记录流程

```
1. 家属登录（用老人手机号或家属账号）
   ↓
2. 进入"我的"页面
   ↓
3. 点击"查看记录"
   ↓
4. 选择记录类型（对话/提醒/应急）
   ↓
5. 查看历史记录
```

## 9. 安全与合规

### 9.1 隐私保护
- 用户信息加密存储
- 对话记录仅保存90天
- 不对外分享用户数据
- 支持用户删除数据

### 9.2 医疗合规
- 明确声明：不做医疗诊断
- 应急情况优先建议就医
- 健康信息仅供参考
- 提醒用户及时就医

### 9.3 数据安全
- 定期数据备份
- 访问权限控制
- 操作日志记录
- 敏感信息脱敏
